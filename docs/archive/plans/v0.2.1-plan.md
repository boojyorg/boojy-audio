## Phase 2: Bug Fixes (3 items)

### 2.1 Fix "New Project" dialog on fresh load
- **File:** `ui/lib/screens/daw/mixins/daw_project_mixin.dart` + `ui/lib/screens/daw_screen.dart`
- **Root cause:** `UndoRedoManager` is a singleton â€” on hot restart, `canUndo` returns true from prior session state, triggering the confirmation dialog even though there's nothing to lose.
- **Fix (two changes):**
  1. In `daw_screen.dart` initState area (~line 119): add `undoRedoManager.clear()` after adding the listener â€” clean slate on every start
  2. In `daw_screen.dart` start screen handler (~line 3207): when handling `StartScreenAction.newProject`, call `_executeNewProject()` directly instead of `newProject()` â€” bypass confirmation from start screen since there's nothing to lose

### 2.2 Fix Master track mixer overflow (0.358px)
- **File:** `ui/lib/widgets/track_mixer_strip.dart`
- **Change:** Add `clipBehavior: Clip.hardEdge` to the Container at ~line 1888. This masks the fractional-pixel overflow from the `rowHeight` division-by-2 calculation without changing any layout math.

### 2.3 Fix toolbar tools centering
- **File:** `ui/lib/widgets/editor_panel.dart`
- **Root cause:** The tools `Center` widget at line 388 is a non-Positioned child of the Stack. In Flutter's Stack, non-Positioned children get loose constraints but aren't guaranteed to center across the full width.
- **Fix:** Wrap the `Center(child: Row(...))` in `Positioned.fill(...)`:
  ```dart
  Positioned.fill(
    child: Center(
      child: Row(mainAxisSize: MainAxisSize.min, children: [...tools...]),
    ),
  ),
  ```
- **Note:** Removing the Piano toggle (1.3) shrinks the right side which may help. Verify centering after 1.3 first, apply this fix if still off.
