# v0.1.8 Release Plan

## Context

v0.1.7 completed Phases 1-4 (#1-#9) of the UI review checklist (design tokens, unified button, icons, typography, animation constants, cluster wells, mixer density, empty states). v0.1.8 tackles **#11 Plugin-as-Instrument redesign** (first priority), **#10 First-run tooltip tour**, **polish items** (min window size, hardcoded colors, debug cleanup), and **bug fixes** via defensive QA.

## 1. Plugin-as-Instrument Redesign (#11)

Third-party VST3 instruments and built-in Boojy instruments in the editor panel.


#### Float Behaviour

```
  CLICK [↗ Float]:
  1. Plugin GUI moves from panel to floating window (200ms transition)
  2. Floating window appears at default position (centred on screen)
     or last remembered position for this plugin
  3. Bottom panel auto-switches to Effects tab
  4. Toggle button changes from [↗ Float] to [↓ Embed]
  5. Preset nav [◀][▾][▶] stays in Row 1, still controls the plugin
  6. Float preference saved to localStorage for this plugin ID

  CLICK [↓ Embed]:
  1. Floating window closes (200ms fade)
  2. Plugin GUI returns to bottom panel
  3. Tab switches back to instrument tab
  4. Toggle button changes from [↓ Embed] to [↗ Float]
  5. Embed preference saved

  CLOSE FLOATING WINDOW ✕:
  Same as clicking [↓ Embed]
  Plugin returns to embedded in panel

  WINDOW POSITION:
  Persisted per-plugin in localStorage.
  Next time this plugin is floated, window opens at same position.
```

### 1.7 Plugin Reset

```
  Default State Capture:
  1. Plugin is added to track → DAW calls getState() immediately
  2. Binary state blob stored in memory as defaultState
  3. User tweaks parameters, browses presets, etc.
  4. User opens preset dropdown, clicks "↺ Reset to Default"
  5. DAW calls setState(defaultState) on the plugin
  6. Plugin reverts to its initial loaded state
  7. Preset name updates to "- Init -" or whatever the default was

  When defaultState Is Captured:
  On first load:    Plugin is instantiated → immediate getState()
  On preset load:   NOT recaptured. defaultState is always the
                    very first state when the plugin was added.
                    "Reset" means "go back to when I first added this."

  If user wants to reset to a specific preset, they use the
  preset browser to navigate to it. Reset is specifically for
  "start over from scratch."
```
