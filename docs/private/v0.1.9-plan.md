# v0.1.9 Release Plan

## Context

v0.1.8 completed the Plugin-as-Instrument redesign (embedded plugin GUIs, float/embed toggle, tab switching lifecycle), first-run tour, and polish items. v0.1.9 tackles the **Combined Device Chain** вҖ” merging the Instrument and Effects tabs into a single horizontal chain view, improving built-in effect UIs, and polishing the editor panel.

The current effects chain (`fx_chain_view.dart`) uses 220px cards in a `ReorderableListView`. The new design replaces both the Instrument tab and Effects tab with a unified chain: instrument device first, then effects, all visible in one scrollable row. This matches Ableton's device chain concept but with full native plugin GUIs.

### 3.4 Device Sizing

```
Built-in effect:       140-180px wide, full chain height
Built-in instrument:   variable width, full chain height
Third-party plugin:    scales to fit chain height uniformly
                       (same setBoundsSize technique as embedded view)
                       width = nativeW * scale where scale = chainHeight / nativeH
                       Never scales UP past 1:1
[+] button:            40px wide, full chain height
```

### 3.5 Plugin Scaling in Chain Box

```
The plugin GUI scales uniformly to fit the chain box height:

  scale = min(chainHeight / nativeH, 1.0)
  displayW = nativeW * scale
  displayH = nativeH * scale

Example: Serum 2 (1190x740) in a 400px tall chain:
  scale = 400 / 740 = 0.54
  displayW = 1190 * 0.54 = 643px
  displayH = 400px (fills height)

The device box width = displayW. No vertical scroll.
No expand/zoom вҖ” use Float for full-size editing.
```
